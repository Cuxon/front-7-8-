# Практические занятия 7–8
## Фронтенд и бэкенд разработка | ИПТИП | 4 семестр 2025/2026

Серверное приложение на Node.js с аутентификацией через bcrypt и JWT, а также CRUD для товаров.

---

## Запуск

```bash
npm install
node index.js
```

Сервер: `http://localhost:3000`
Swagger UI: `http://localhost:3000/api-docs`

---

## Стек

- **Node.js** + **Express** — сервер
- **bcrypt** — хеширование паролей с солью (10 раундов)
- **jsonwebtoken** — выдача и валидация JWT (access-токен, TTL 15 минут)
- **nanoid** — генерация уникальных id
- **swagger-jsdoc** + **swagger-ui-express** — интерактивная документация API

---

## Практика 7 — Базовые методы аутентификации

Реализована регистрация и вход пользователя с хешированием пароля через bcrypt.

### Маршруты

| Метод | Маршрут | Описание |
|-------|---------|----------|
| POST | `/api/auth/register` | Регистрация пользователя |
| POST | `/api/auth/login` | Вход в систему |
| POST | `/api/products` | Создать товар |
| GET | `/api/products` | Получить список товаров |

### Сущность «Пользователь»

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный идентификатор (nanoid) |
| `email` | string | Логин пользователя |
| `first_name` | string | Имя |
| `last_name` | string | Фамилия |
| `password` | string | Хеш пароля (bcrypt) |

### Сущность «Товар»

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный идентификатор (nanoid) |
| `title` | string | Название |
| `category` | string | Категория |
| `description` | string | Описание |
| `price` | number | Цена |

## Практика 8 — JWT-аутентификация

Добавлена выдача JWT при входе и защита маршрутов через middleware.

### Что добавлено

- `POST /api/auth/login` теперь возвращает `accessToken` (JWT, TTL 15 мин)
- `authMiddleware` — проверяет заголовок `Authorization: Bearer <token>`, валидирует токен через `jwt.verify()`
- Защищённый маршрут `GET /api/auth/me` — возвращает данные текущего пользователя

### Защищённые маршруты (требуют токен)

| Метод | Маршрут | Описание |
|-------|---------|----------|
| GET | `/api/auth/me` | Текущий пользователь |
| GET | `/api/products/:id` | Получить товар по id |
| PUT | `/api/products/:id` | Обновить товар |
| DELETE | `/api/products/:id` | Удалить товар |

### Как использовать токен

1. Выполнить `POST /api/auth/login` — получить `accessToken`
2. Передавать в заголовке каждого защищённого запроса:
```
Authorization: Bearer <accessToken>
```
